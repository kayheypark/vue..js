beforeCreate()- 뷰인스턴스 생성되고나서 new Vue()

created() - 뷰 인스턴스가 생성되자마자 뷰 데이터에 접근할 수 있도록 created(), 서버에 데이터를 요청하여 받아오는 로직 수행

beforeMount() - 부착전, #app 에 뷰인스턴스를 부착하기 전에 호출됨 

Mounted() - 뷰 인스턴스를 화면에 부착 후, trmplate속성에 지정한 마크업 속성을 render()함수 즉 자바스크립트로 화면의 돔을 긜는 함수로 변환한 후 el 속성에 지정한 화면요소(돔) #app 에 뷰 인스턴스를 부착


beforeUpdate() - 화면 변경전 변경 예정인 새 데이터에 접근 가능 시점
updated() - 데이터가 변경되고 나서 가상돔으로 다시 화면을 그리고나면 실행되는 시점

beforeDestory()
updated() - 뷰 인스턴스의 데이터를 삭제하는 시점
destroyed() - 뷰 인스턴스 소멸후

vue 특징
1. 양방향 데이터 바인딩 (앵귤러의장점)
:화면에 표시되는 값과 프레임워크의 모델Model 데이터 값이 동기화되어 한쪽이 변경되면 다른 한쪽도 자동으로 변경되는 것.

2. 단방향 데이터 흐름 (react의장점)
:컴포넌트 간에 데이터를 전달할 때 항상 상위 컴포넌트에서 하위 컴포넌트 한 방향으로만 전달하게 프레임워크가 구조화 되어 있다.
